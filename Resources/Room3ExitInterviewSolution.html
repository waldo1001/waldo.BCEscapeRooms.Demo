Solution: Room 3 - Exit Interview

The exit interview is an automated test that verifies the results of Room 1 and Room 2. There is nothing new to configure — just ensure the previous rooms were completed correctly.

Pass the Exit Check

What the Test Checks

The automated verification runs two assertions:

  1. Company Information — Name must be iFacto (from Room 1)
  2. Contact — A Contact with Company Name Directions Partner must exist (from Room 2)

If the Test Fails

The error message tells you exactly what is missing. Go back and fix it:

  • If the company name is wrong: open Company Information and correct it
  • If the contact is missing: open Contacts and create one with Company Name Directions Partner

Why This Matters:

  • This task uses the test codeunit validation pattern
  • A codeunit with Subtype = Test runs when Update Status is clicked
  • The test performs multi-step validation across multiple tables — something that simple polling or event subscribers cannot do as cleanly
  • The framework's Task Validation Test Runner handles execution: it catches errors and translates them into pass/fail results
  • SelectLatestVersion() ensures the task reads the test result from the database, not from NST cache

The test codeunit pattern is the most powerful validation option. It can test anything a normal AL test can test — table data, page availability, complex business logic, even UI interactions via page test actions.

Verification

  • Task shows Completed
  • Room 3 closes, and the venue is completed
  • The venue's Stop DateTime is set and telemetry is logged
